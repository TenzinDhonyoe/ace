# Ace

Open-source infrastructure for AI-generated interactive study content.

Drop your lecture PDFs on Claude. Get back a deployable study site with **live physics sandboxes**, not just flashcards.

Reference output: [bme804-review.vercel.app](https://bme804-review.vercel.app/#act) — built by hand, regenerated by Ace.

---

## Why

Every student studies by re-reading lectures and asking ChatGPT to "quiz me." Nobody auto-generates **working physics/math/bio visualizations** from lecture content. That's the whitespace Ace fills.

This repo ships three things:

1. **`@ace/components`** — a framework-agnostic library of 5 widgets (sliders wired to equations, flashcards, MCQs, diagram labels, concept cards). Works in plain HTML, React, Vue, anything.
2. **`@ace/template`** — a renderer that turns a `site.config.json` into a deployable HTML file.
3. **`ace-skill`** — a Claude skill that, given lecture PDFs + exam structure, emits the `site.config.json` and invokes the renderer.

The component library is the real IP. The generator is the flagship demo.

## For students (the main path)

Prereqs: [Bun](https://bun.sh) and [Claude Code](https://claude.com/claude-code) installed.

```bash
# 1. Clone the repo (or use GitHub's "Use this template")
git clone https://github.com/tenzindhonyoe/ace my-bme804-review
cd my-bme804-review

# 2. Install widget deps
bun install

# 3. Drop your lecture PDFs into ./inputs/
#    (optional: write a short inputs/exam-structure.md)

# 4. Open Claude Code and run the skill
claude
> /ace-review
```

The skill auto-loads from `.claude/skills/ace-review/SKILL.md` in this repo.
It reads every PDF, builds `./output/site.config.json`, renders `./output/index.html`,
and reports done. No manual widget wiring.

Open `output/index.html` in a browser, or drag `output/` onto Vercel to deploy.

**Edit without regenerating:** hand-edit `output/site.config.json`, then:
```bash
bunx @ace/template output/site.config.json -o output/
```

## For developers

**Use the widget library in any site:**

```html
<link rel="stylesheet" href="https://unpkg.com/@ace/components/styles.css">
<div id="mount"></div>
<script type="module">
  import { SliderSandbox } from "https://unpkg.com/@ace/components";
  new SliderSandbox().mount(
    document.getElementById("mount"),
    {
      id: "ohms-law",
      title: "Ohm's law",
      variables: [
        { key: "V", label: "Voltage", unit: "V", min: 0, max: 12, step: 0.1, initial: 5 },
        { key: "R", label: "Resistance", unit: "Ω", min: 1, max: 1000, step: 1, initial: 100 }
      ],
      outputs: [
        { key: "I", label: "Current", unit: "A", format: "si" },
        { key: "P", label: "Power", unit: "W", format: "si" }
      ],
      compute: "const I = vars.V / vars.R; return { I, P: vars.V * I };"
    },
    { siteId: "my-page" }
  );
</script>
```

That's it. No React, no build step, no framework lock-in. ~10KB gzipped for the entire library.

## Architecture

```
packages/
  ace-components/         # the widget library — 5 types
    src/
      slider-sandbox/     # the moat: generic sliders + compute + SVG
      flashcard/          # blur-to-reveal deck with mastery tracking
      mcq-quiz/           # self-test with persistent answers
      diagram-label/      # numbered hotspots + label pool
      concept-card/       # single quick-check retrieval
      lib/                # shared: storage, validate, widget base, format
    schemas/              # JSON Schema: site.config + per-widget (imported by Claude)
    registry.js           # widget registry — contract with the generator
    styles.css            # shared CSS (BEM-prefixed with ace-)
  ace-template/           # site.config.json → HTML renderer + CLI
ace-skill/                # Claude skill (markdown) for non-dev students
eval/                     # generator quality harness (3 golden cases)
apps/
  gallery/                # landing page with live widget embeds
```

**Key design decisions** (locked during planning, see `DESIGN.md`):

- Plain-JS classes with `mount(el, props)` / `destroy()` — no React dep, works everywhere
- Per-widget JSON Schema — machine-readable by Claude, runtime-validated at mount, TS types auto-generated
- Explicit `registry` object — grep-able, adding a widget = one import + one key
- BEM-prefixed CSS (`.ace-*`) — no Shadow DOM in Phase 1, site authors can restyle
- `site.config.json` is the contract — generator emits it, renderer reads it, both are swappable
- Storage namespaced `ace:<siteId>:<widgetId>:<key>` — no cross-site collisions
