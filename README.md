# Ace

> Drop your lecture PDFs on Claude. Get back a deployable study site with **live physics sandboxes**, not just flashcards.

Every student studies by re-reading slides and asking ChatGPT to "quiz me." Nobody auto-generates **working visualizations** from lecture content — sliders wired to real equations, diagram labels from real figures, MCQs built from real misconceptions. Ace does.

Reference output: [bme804-review.vercel.app](https://bme804-review.vercel.app/#act) — built by hand, regenerated by Ace.

> **Status:** pre-0.3. The npm packages aren't published yet, so `bunx` / `unpkg` commands don't work. Clone the repo and follow the Quick start below. See [PUBLISHING.md](PUBLISHING.md) for the publish plan.

**Who this is for:**
- **Students** cramming for a midterm who want a real study site, not another flashcard app
- **TAs and instructors** who want to give their class an interactive review they didn't have to build by hand
- **Ed-tech builders** who want a framework-agnostic widget library (5 types, ~10KB gzipped) to drop into their own stack

## Quick start

**Requirements:** [Bun](https://bun.sh/) 1.3+, [Claude Code](https://claude.com/claude-code), [Git](https://git-scm.com/).

Open Claude Code in any directory and paste this. Claude does the rest.

> Set up Ace for me: run **`git clone https://github.com/tenzindhonyoe/ace my-exam-review && cd my-exam-review && bun install`**, then `cd` into `my-exam-review` for the rest of this session. After install completes, confirm `bun --version` works, check that `packages/ace-template/bin/cli.js` exists, and show me the `inputs/` folder. Then tell me to drop my lecture PDFs into `./inputs/` (and optionally write a short `inputs/exam-structure.md` with Course, Format, and Sections), and that I should run `/ace-review` when I'm ready. Do not run `/ace-review` yourself — wait for me.

Then:

1. Drop your lecture PDFs into `./inputs/`
2. In Claude Code, run `/ace-review`
3. Open `./output/index.html` — or drag `output/` onto Vercel
4. Share the URL with your class

That's the whole loop. No manual widget wiring.

**Prefer to do it yourself?** Every step is spelled out below.

## Install — step by step

**Requirements:** [Bun](https://bun.sh/) 1.3+, [Claude Code](https://claude.com/claude-code), [Git](https://git-scm.com/)

### Step 1: Clone the repo

```bash
git clone https://github.com/tenzindhonyoe/ace my-exam-review
cd my-exam-review
bun install
```

`bun install` is a one-time step. It installs the widget library dependencies that the renderer needs.

### Step 2: Add your lecture PDFs

Drop every PDF for the exam into `./inputs/`. Name them however you want.

```bash
cp ~/Downloads/lecture-*.pdf inputs/
```

**Optional but recommended:** write a short `inputs/exam-structure.md` describing the exam. It helps the generator focus. If you skip it, `/ace-review` infers from the PDF contents.

```markdown
Course: BME 804 — Biomedical Instrumentation
Format: MCQ-heavy, 90 minutes, open notes
Sections:
- Signal conditioning (amplifiers, filters)
- ADCs and sampling
- Biopotential electrodes
- Safety and isolation
```

### Step 3: Run the skill

```bash
claude
```

Then in Claude Code, type:

```
/ace-review
```

The skill auto-loads from `.claude/skills/ace-review/SKILL.md`. It reads every PDF, composes `./output/site.config.json`, renders `./output/index.html`, and reports done. Takes 2-5 minutes depending on how many lectures you dropped in.

### Step 4: Open your site

```bash
open output/index.html
```

Or deploy it. Three one-command options:

```bash
# Vercel
npx vercel --prod output/

# Netlify
npx netlify deploy --prod --dir=output/

# Or drag the output/ folder onto https://app.netlify.com/drop
```

Share the URL. Your classmates don't need Bun, Claude Code, or any install — it's just HTML + JS.

### Step 5: Edit without regenerating

Hand-edit `output/site.config.json`, then re-render:

```bash
bun packages/ace-template/bin/cli.js output/site.config.json -o output/
```

No need to re-read PDFs. Cheap and fast.

## For developers

Three packages, all MIT:

1. **`ace-components`** — framework-agnostic widget library (5 types, plain JS classes, ~10KB gzipped). Works in static HTML, React, Vue, anywhere.
2. **`ace-template`** — renderer that turns a `site.config.json` into a deployable HTML file.
3. **`.claude/skills/ace-review/`** — the Claude skill that orchestrates the pipeline.

> Until the packages hit npm, use the local source (`packages/ace-components/`) or copy the built bundle from `output/` after running `/ace-review` once.

**Once published, using the widget library in any site looks like this:**

```html
<link rel="stylesheet" href="https://unpkg.com/ace-study-components/styles.css">
<div id="mount"></div>
<script type="module">
  import { SliderSandbox } from "https://unpkg.com/ace-study-components";
  new SliderSandbox().mount(
    document.getElementById("mount"),
    {
      id: "ohms-law",
      title: "Ohm's law",
      variables: [
        { key: "V", label: "Voltage", unit: "V", min: 0, max: 12, step: 0.1, initial: 5 },
        { key: "R", label: "Resistance", unit: "Ω", min: 1, max: 1000, step: 1, initial: 100 }
      ],
      outputs: [
        { key: "I", label: "Current", unit: "A", format: "si" },
        { key: "P", label: "Power", unit: "W", format: "si" }
      ],
      compute: "const I = vars.V / vars.R; return { I, P: vars.V * I };"
    },
    { siteId: "my-page" }
  );
</script>
```

No React, no build step, no framework lock-in.

## Architecture

```
packages/
  ace-components/         # widget library — 5 types
    src/
      slider-sandbox/     # the moat: generic sliders + compute + SVG
      flashcard/          # blur-to-reveal deck with mastery tracking
      mcq-quiz/           # self-test with persistent answers
      diagram-label/      # numbered hotspots + label pool
      concept-card/       # single quick-check retrieval
      lib/                # shared: storage, validate, widget base, format
    schemas/              # JSON Schema: site.config + per-widget (imported by Claude)
    registry.js           # widget registry — contract with the generator
    styles.css            # shared CSS (BEM-prefixed with ace-)
  ace-template/           # site.config.json → HTML renderer + CLI
.claude/skills/ace-review/ # Claude skill that orchestrates the pipeline
eval/                     # generator quality harness (5 golden cases)
inputs/                   # user drops PDFs here
output/                   # generated site lands here (gitignored)
```

**Key design decisions** (see [docs/architecture.md](docs/architecture.md) for deep rationale):

- Plain-JS classes with `mount(el, props)` / `destroy()` — no React dep, works everywhere
- Per-widget JSON Schema — machine-readable by Claude, runtime-validated at mount, TS types auto-generated
- Explicit `registry` object — grep-able, adding a widget = one import + one key
- BEM-prefixed CSS (`.ace-*`) — no Shadow DOM in Phase 1, site authors can restyle
- `site.config.json` is the contract — generator emits it, renderer reads it, both are swappable
- Storage namespaced `ace:<siteId>:<widgetId>:<key>` — no cross-site collisions

## Troubleshooting

**`/ace-review` isn't available in Claude Code?** Make sure you opened Claude Code from the repo root (`cd my-exam-review && claude`). The skill auto-loads from `.claude/skills/ace-review/SKILL.md` in the current directory.

**Skill says "inputs/ is empty"?** Drop PDFs into `./inputs/`, then run `/ace-review` again in the same session.

**Renderer rejects `site.config.json`?** Read the error, fix the specific field in `output/site.config.json`, then re-run:
```bash
bun packages/ace-template/bin/cli.js output/site.config.json -o output/
```

**PDF won't read (scanned image, non-OCR'd)?** The skill skips that PDF and proceeds with the rest. For best results, use text-based PDFs or run them through an OCR tool first.

**Bundle budget.** Target is 15KB gzipped (currently ~10.6KB). CI enforces a 32KB raw ceiling. Regressions land loudly on purpose.
